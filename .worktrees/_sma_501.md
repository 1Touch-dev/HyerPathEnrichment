# Fix: Multilogin 501 "Unsupported method ('POST')" Error

## Problem

Campaigns were failing with this error during Multilogin signin:

```
Error during login: <!DOCTYPE HTML>
<html lang="en">
    <head>
        <meta charset="utf-8">
        <title>Error response</title>
    </head>
    <body>
        <h1>Error response</h1>
        <p>Error code: 501</p>
        <p>Message: Unsupported method ('POST').</p>
        <p>Error code explanation: 501 - Server does not support this operation.</p>
    </body>
</html>
```

**Impact**:
- 9 out of 10 campaigns failed
- 4 campaigns didn't process
- Only 1 campaign ran successfully

## Root Cause

HTTP(S) proxy environment variables were routing requests to `https://api.multilogin.com` through a local proxy server that **does not support POST requests**.

**Environment variables causing the issue**:
```bash
HTTPS_PROXY=http://127.0.0.1:56494
HTTP_PROXY=http://127.0.0.1:56494
NO_PROXY=127.0.0.1,::1,localhost  # api.multilogin.com NOT included!
```

The proxy at port 56494 appears to be a simple Python `http.server` or similar that only supports GET requests, hence the 501 error.

## Solution

### Code Changes

Modified all Multilogin API calls to **explicitly bypass proxies** using:

```python
NO_PROXY_FOR_MULTILOGIN = {"http": None, "https": None}

# All requests.post() and requests.get() calls now include:
requests.post(url, ..., proxies=NO_PROXY_FOR_MULTILOGIN)
```

### Files Modified

1. **`plugins/shared/core/multilogin.py`** (main file)
   - Added `NO_PROXY_FOR_MULTILOGIN` constant
   - Updated `create_ssl_session()` to bypass proxy by default
   - Updated `signin_multilogin()` to bypass proxy on all attempts
   - Updated `/user/refresh_token` call to bypass proxy

2. **`plugins/shared/core/multilogin_safe.py`**
   - Same changes as above

3. **`plugins/shared/core/multilogin_relaxed.py`**
   - Same changes as above

4. **`plugins/shared/core/session.py`**
   - Added proxy bypass to signin call

5. **`multilogin_proxy_refresh.py`**
   - Added proxy bypass to all Multilogin API calls

### Alternative Solutions (Environment-Based)

If you prefer not to modify code, you can fix this at the environment level:

**Option 1**: Add `api.multilogin.com` to NO_PROXY:
```bash
export NO_PROXY=127.0.0.1,::1,localhost,api.multilogin.com
```

**Option 2**: Unset proxy variables for Celery workers:
```bash
unset HTTP_PROXY HTTPS_PROXY
```

## Technical Details

The 501 error occurs because:

1. Python's `requests` library respects `HTTP_PROXY`/`HTTPS_PROXY` environment variables
2. When these are set, all HTTP(S) requests are routed through the proxy
3. The proxy server at `127.0.0.1:56494` only implements `GET` (like Python's `http.server`)
4. Multilogin's `/user/signin` requires `POST`, which the proxy rejects with 501

By setting `proxies={"http": None, "https": None}`, we tell requests to bypass any environment proxy configuration and connect directly to the target URL.
